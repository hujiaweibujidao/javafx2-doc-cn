Deploying JavaFX Applications

Troubleshooting

  • ←上一篇

Use the following checklist if you have trouble running applications after you package them.

  1. Verify that you have a supported environment. See the System Requirements document for your JavaFX SDK release at
    http://docs.oracle.com/javafx/release-documentation.html

  2. Check the release notes for known issues at
    http://docs.oracle.com/javafx/release-documentation.html

  3. Check that the JavaFX Runtime is installed. Go to Control Panel, then select Add/Remove Programs and verify that JavaFX 2.n appears in the list of installed applications.

  4. Ensure the JavaFX samples run properly. Download the JavaFX samples zip file at

    http://www.oracle.com/technetwork/java/javafx/downloads/index.html

    Double-click the JAR file, the JNLP file, and the HTML file for at least one sample to ensure it runs correctly.

    • If JavaFX samples do not run correctly, try running a standard Java applet or Web Start application to ensure the JDK or JRE is functioning properly on your system.

  5. If a Clean and Build in the NetBeans IDE fails to produce the three files (JAR, JNLP, and HTML), or if double-clicking some or all of the files results in an error, check that the JavaFX platform is enabled properly in NetBeans. See Setting Up NetBeans IDE With JavaFX at
    http://netbeans.org/kb/docs/java/javafx-setup.html

  6. If running javafxpackager at the command line results in many errors that look like it is not parsing the source code correctly, or if you get the error String index out of range: -1, try setting a JAVAFX_HOME environment variable in Windows, which points to the \bin directory of your JavaFX SDK installation. Example for a default installation:

    Variable: JAVAFX_HOME
    Value: C:\Progra~1\Oracle\JAVAFX~1.0SD\bin

  7. If you make a change to the javafxpackager command at the command line but the resulting output files fail to reflect the change, clear the Java cache. Open Java Control Panel in Windows by opening Windows Control Panel, then choose Java. In Java Control Panel, in the Temporary Internet Files section, click Settings, then click Delete Files. Select Cached Applications and Applets and click OK.

  8. If the problem is specific to launching a JavaFX application in the browser or from the browser, see the Java SE 7 Desktop Troubleshooting Guide for hints on how to troubleshoot generic Java plug-in and Java Web Start problems.

  9. If you still have unresolved issues, go to the JavaFX Forum at
    https://forums.oracle.com/forums/forum.jspa?forumID=1385
